# Zenn 自動記事生成プロジェクト

## プロジェクト概要
Claude Code を使ってAI・データサイエンス系のZenn記事を生成し、GitHubにpushするプロジェクト。

## 記事生成の手順

「新しい記事を1本生成してpushしてください」と頼まれたら以下を実行する:

1. **「執筆済み記事」セクションを確認し、重複しないトピックを選ぶ**
2. 下記カテゴリ・トピックリストから1つ選ぶ
3. `articles/` に Zennフォーマットの `.md` ファイルを作成する
4. **記事ファイルを1本ずつ別のcommitでpushする**
   ```
   git add articles/{ファイル名}.md && git commit -m "docs: タイトル" && git push
   ```
   ⚠️ Zennの無料プランは1日2本までしか新規デプロイできない。1日1本ペースが安全。
5. **「執筆済み記事」セクションに書いた記事を追記する**
6. git push が完了したら以下を両方実行する:
   **Qiitaにも投稿する（自動）:**
   ```
   python scripts/post_to_qiita.py articles/{作成したファイル名}.md
   ```
7. X投稿用テキストを表示する:
   【X投稿はパターンA（問題提起型）で作成すること】
   構成: ①失敗例・状況 → ②結果・問題点 → ③なぜそうなるか → ④記事の価値👇 → URL → ハッシュタグ
   ```
   python scripts/post_to_x.py articles/{作成したファイル名}.md
   ```
   表示されたテキストをそのままユーザーに見せる（ユーザーがXにコピペする）

## 記事カテゴリ

**⚠️ 現在のテーマ：統計検定データサイエンティストエキスパート対策**
すべての記事を試験範囲に沿ったトピックで執筆する。Pythonコードで概念を実装し、試験で問われる理論と実践を両方カバーすること。

### 📊 確率・統計の基礎
- 確率分布の使い分け（正規・二項・ポアソン・t・F・χ²分布など）
- 最尤推定（MLE）の仕組みと実装
- モーメント法・推定量の性質（不偏性・一致性・有効性）
- 中心極限定理と大数の法則
- 情報量基準（AIC・BIC）

### 🔬 統計的推定・検定
- 区間推定と信頼区間の正しい解釈
- 仮説検定の体系（t検定・χ²検定・F検定・Mann-Whitney検定）
- 多重比較問題とBonferroni補正・FDR制御
- 検出力分析とサンプルサイズ設計
- ノンパラメトリック検定の使い所

### 📈 回帰・多変量解析
- 重回帰分析と多重共線性の診断・対処
- 一般化線形モデル（ロジスティック回帰・ポアソン回帰）
- 主成分分析（PCA）の数学的背景と実装
- 因子分析・クラスター分析
- 判別分析・カーネル法

### ⏱️ 時系列・ベイズ
- 時系列の定常性・自己相関・ARIMAモデル
- 状態空間モデルとカルマンフィルタ
- ベイズ推論・MCMC法の実装
- 階層ベイズモデル
- 因果推論（傾向スコア・DID・回帰不連続）

### 🤖 機械学習・評価
- 偏差-分散トレードオフと正則化
- 交差検証とモデル選択
- 評価指標の使い分け（ROC・PR曲線・F値）
- アンサンブル学習（バギング・ブースティング）
- 次元削減と特徴量エンジニアリング

## Zenn 記事フォーマット

ファイル名: `YYYYMMDD-{英語スラッグ}.md`（例: `20250604-data-scientist-growth.md`）

```markdown
---
title: "記事タイトル（日本語OK）"
emoji: "📊"
type: "tech"  # tech: 技術記事 / idea: 意見・考察
topics: ["python", "datascience", "statistics"]  # 最大5個
published: true
---

## はじめに
（背景・なぜこれが重要か）

## 本題
（複数セクション、コード例を含む）

## まとめ
（要点整理・次のステップ）
```

### typeの使い分け
- `tech`: 実装・ハンズオン系、コードが主体
- `idea`: キャリア・考察・ニュース解説系

### 執筆ルール
- 日本語で執筆
- 対象読者: Python中級者（1〜3年目エンジニア）
- 文字数: 3000〜5000文字
- techの場合はPythonコード例を必ず含める
- **年号をタイトルや本文に入れない。「〇〇年版」「〇〇年最新」は使わない**
- **情報が陳腐化しにくい「概念・実践・使い方」ベースで書く**
- 悪い例：「2026年最新LLM比較」「2025年版データサイエンス入門」
- 良い例：「Claude・GPT・Gemini 用途別の選び方」「現場で使えるデータ分析の基本」

### 統計検定DS対策記事の構成ルール（必須）
各トピックについて以下の3点を必ずセットで書くこと:
1. **数式**：定義・定理をLaTeX形式（$...$）で記載する。期待値・分散・確率密度関数など試験で問われる式を省略しない
2. **いつ使うか**：どんなデータ・状況でこの手法を選ぶか。使う条件・前提も明記する
3. **Pythonコード**：数式をコードで実装し、結果を確認できるようにする

構成例:
```
## ○○とは
（直感的な説明）

### 数式
$$数式$$
（各記号の意味を説明）

### いつ使うか
- 条件A のとき
- データがXXの形のとき

### Pythonで実装
（コード）
```
